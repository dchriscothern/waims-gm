Capture order:

1. `board-overview.png`

   - use multiple seeded files
   - keep the board sorted and readable

2. `dossier-view.png`
   - show the `Player Dossier` tab
   - capture the top executive brief, diagnostic snapshot, and the start of the file

3. `compare-view.png`
   - show the `Compare` tab
   - capture the comparison summary, decision snapshot, and verdict cards

4. `workflow-actions.png`

   - or capture export actions and mode focus controls if that tells the story better

5. `demo-walkthrough.gif`
   - quick board scan
   - open one dossier
   - switch to compare
   - show export action

Best capture rules:

- use local demo mode so the app is stable and fast
- use the same warm visual theme consistently
- crop tightly enough that the product, not the browser chrome, is the focus
- prefer one clean story per screenshot instead of trying to show everything at once
- if a screenshot needs annotation, add it later outside the app rather than cluttering the UI
